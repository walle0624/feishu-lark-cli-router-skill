# Contributing

Thanks for improving this skill.

## Scope

Good contributions usually improve one of these areas:

- Feishu/Lark intent routing
- `lark-cli` onboarding and recovery flows
- safer public-sharing defaults
- clearer trigger language for Codex

## Guidelines

- keep the skill focused on Feishu/Lark workflows
- prefer reusable, shareable instructions over machine-specific hacks
- do not commit secrets, tokens, auth files, or personal account data
- avoid hard-coded absolute home-directory paths
- update `README.md` when user-facing behavior changes

## Pull Requests

When opening a pull request, please describe:

- the user problem you are fixing
- the skill behavior before the change
- the skill behavior after the change

Concrete examples are especially helpful for trigger wording and onboarding flows.
